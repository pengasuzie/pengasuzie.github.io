How It Works

The full chain, one command

Combines strategic analysis and text-to-voice into a single workflow. Reads your project context, analyzes the transcript with product-aware intelligence, converts the analysis to speech-friendly text, generates an MP3, and copies it to Dropbox. You get a written analysis AND an audio briefing.

  • Project-aware analysis (reads CLAUDE.md, docs, git log)
  • Identifies participants from transcript
  • Cross-references competitors with existing intel
  • Produces analysis.md + analysis-voice.txt + analysis-audio.mp3
  • Auto-copies to Dropbox for mobile listening
  • Works after /voice-to-text or with any transcript

What Gets Produced

Three files, one workflow

The written analysis (analysis.md) is a full strategic breakdown. The voice text (analysis-voice.txt) strips markdown and adds spoken cues. The audio (analysis-audio.mp3) is the voice text converted to speech, ready for your commute.

analysis.md (written) └── Product Context (auto-gathered) ├── Comprehensive Summary ├── Strategic Analysis │ └── Win Probability │ ├── Feature-by-Feature Fit │ ├── Development Velocity │ └── Strategic Angles ├── The Hard Truths │ └── Competitive Landscape │ ├── Red Flags │ └── Next Actions └── Roadmap Impact Assessment analysis-voice.txt (TTS-optimized) Markdown stripped, tables → prose, acronyms expanded, spoken transitions. Project Context Awareness

Grounded in what you know

The analysis is only as good as the context. The skill reads everything available — CLAUDE.md for product state, git log for development velocity, docs for competitor intel, and memory for prior conversations. The result is advice grounded in reality, not generic strategy.

  • .claude/CLAUDE.md — product identity, stage, roadmap, milestones
  • git log — recent commits, development velocity, what just shipped
  • docs/private/ — competitor analysis, strategic documents
  • notes/ — prior meeting summaries, strategy sessions
  • memory/ — persistent context from prior conversations
